# Product

## Register

product

## Users

Developers and AI-agent power users who want Oh My Pi's terminal-grade coding workflow in a durable desktop shell: multi-session workspaces, permission controls, model selection, source/tool management, and long-running agent work that can be monitored visually.

## Product Purpose

Oh My Pi Desktop turns OMP from a terminal-first coding agent into a mature desktop product while preserving its core strengths: IDE-aware tooling, model/provider flexibility, structured session orchestration, and explicit user control. Success means the app feels native, reliable, and unmistakably OMP rather than a lightly reskinned Craft fork.

## Brand Personality

Precise, electric, capable. The product should feel like a serious developer instrument with a vivid blue-violet energy layer: more terminal/IDE command deck than generic SaaS dashboard.

## Anti-references

Avoid anonymous grayscale Craft defaults, decorative gradients pasted onto otherwise unchanged UI, generic “AI purple” surfaces, over-branded controls that distract from work, and icons that depend on fragile font glyphs.

## Design Principles

1. Make OMP visible at the system seams: app icon, provider icon, onboarding, default theme, and titlebar should all speak the same language.
2. Keep product UI restrained: blue-violet color should guide current state, selection, focus, and primary actions, not decorate every component.
3. Preserve developer trust: compact density, stable affordances, readable contrast, and predictable platform behavior matter more than spectacle.
4. Prefer robust branded primitives: SVG paths, explicit assets, and semantic theme tokens over font-dependent glyphs or one-off styling.
5. Treat local-first agent work as the hero: the shell should feel engineered around sessions, tools, rules, and model choice.

## Accessibility & Inclusion

Target WCAG AA contrast for text and interactive controls. Do not rely on color alone for state; pair color with labels, icons, or structural cues. Keep motion subtle and state-driven, and preserve usable light and dark modes.
